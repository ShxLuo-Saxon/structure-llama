# StructureLlama

Training and evaluation code for the paper:

> **Structure-Aware Game Music Generation with Similarity-Bucket Conditioning**
> *[Authors redacted for review]* · ISMIR 2026
> [Paper link — to be added after review]

Companion dataset: [`supermario-structure-annotation`](https://github.com/ShxLuo-Saxon/supermario-structure-annotation) (554 structural annotations + 3,304 section pairs).

---

## Overview

Given a context section and a similarity level (`sim:high` / `sim:mid` / `sim:low`), StructureLlama generates a target section whose structural relationship to the context matches the requested level. The backbone is [Moonbeam](https://github.com/guozixunnicolas/Moonbeam-MIDI-Foundation-Model) 839M, fine-tuned with LoRA.

## Data

You need:

1. **Section `.npy` files** — generated by the preprocessing pipeline below from the MXL files available via the companion dataset repo.
2. **`pairs.csv`** — the exact pair index used for the paper (3,304 pairs, train/val/test split, `sim_bucket` labels) is included at [`outputs/pairs.csv`](outputs/pairs.csv). Copy it into the same directory as the `.npy` files.
3. A **Moonbeam 839M checkpoint** (`moonbeam_839M.pt`).

Let `$SEC` be the directory holding the `.npy` files and `pairs.csv`, and `$CKPT` the Moonbeam checkpoint path.

The Moonbeam 839M base checkpoint is available at [HuggingFace](https://huggingface.co/guozixunnicolas/moonbeam-midi-foundation-model/tree/main) (~1.5 GB).

### Preprocessing pipeline

```
NinSheetMusic .mxl files   (via companion dataset repo)
        │
        │  scripts/data_processing/mxl_to_mid.py
        │  Unrolls repeats, D.C./Segno, 2nd endings → flat 120 BPM MIDI
        ▼
processed/mid/*.mid
        │
        │  scripts/data_processing/structure_data_preprocess.py
        │  Tokenises MIDI, slices by annotation section boundaries → one .npy per section
        ▼
processed/npy/sections/*.npy
        │
        │  scripts/data_processing/build_section_pairs.py
        │  Builds directed within-piece pairs, assigns sim_bucket labels
        ▼
processed/npy/sections/pairs.csv
```

> **Note on 28 manually corrected pieces:** complex coda/2nd-ending notation in these pieces required manual MXL edits that cannot be redistributed. Without the fixes, those pieces produce MIDI with incorrect playback order (~5% of dataset). The `fix_log` column in [`outputs/metadata.xlsx`](outputs/metadata.xlsx) documents each correction. `structure_data_preprocess.py` will warn and skip out-of-range sections rather than crash.

## Training

Three model variants are trained from the same base checkpoint:

| Model | Conditioning |
|---|---|
| M1 | correct `sim:bucket` label |
| M2 | no bucket (unconditional baseline) |
| M3 | randomly wrong `sim:bucket` label (ablation) |

```bash
sbatch scripts/slurm/train.sh               # M1
CMODE=m2 sbatch scripts/slurm/train.sh      # M2
CMODE=m3 sbatch scripts/slurm/train.sh      # M3
```

Direct invocation (single GPU):

```bash
python scripts/training/train.py \
    --config_path    scripts/configs/model_config_structure_llama_839M.json \
    --sections_dir   $SEC \
    --pairs_csv      $SEC/pairs.csv \
    --checkpoint     $CKPT \
    --condition_mode m1 --use_lora \
    --lora_rank 8 --lora_alpha 16 \
    --context_length 1024 --batch_size 2 \
    --num_epochs 200 --lr 3e-5 --constant_lr \
    --patience 20 \
    --save_dir runs/m1
```

Edit `<your_account>` and `<your_gpu_partition>` in `scripts/slurm/*.sh` before submitting.

## Evaluation

```bash
# Ctx-sim + GT-sim (reproduces Tables 2 & 3)
python scripts/eval/eval_generation.py \
    --triplet_dir outputs/generated/triplet \
    --sections_dir $SEC --pairs_csv $SEC/pairs.csv \
    --out_csv outputs/eval/results_tests.csv \
    --truncate_to_ctx

# CFG eval
python scripts/eval/eval_cfg.py \
    --cfg_dir outputs/generated/cfg \
    --sections_dir $SEC --pairs_csv $SEC/pairs.csv \
    --out_csv outputs/eval/results_cfg_tests.csv \
    --truncate_to_ctx

# Bucket-threshold analysis
python scripts/eval/within_piece_sim_dist.py --sections_dir $SEC
```

Pre-computed results reproducing all paper tables are in [`outputs/eval/`](outputs/eval/).

The `compound_sim` metric reference implementation: [`scripts/eval/compound_metric.py`](scripts/eval/compound_metric.py).
